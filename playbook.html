The API uses cursor-based pagination (returns `next_cursor` in response) and limits results to 100 per page. Workflow requirements: 1) Make initial request to `/api/v1/contacts` endpoint, 2) Extract records and next_cursor from response, 3) Continue fetching until next_cursor is null, 4) Accumulate all records across all pages, 5) Transform each record to match database schema (map API fields to DB columns), 6) Batch insert into PostgreSQL with upsert logic (update if email exists, insert if new), 7) Log total records fetched and processing time. Handle rate limiting (429 responses with Retry-After header).
💡 EXAMPLE OUTPUT
Function node (init: cursor=null, allRecords=[]) → Loop node → HTTP Request (GET /api/v1/contacts?cursor={{cursor}}&limit=100) → Function (append response.data to allRecords, update cursor = response.next_cursor) → IF (cursor is not null) → True: loop back → False: exit loop → PostgreSQL node (INSERT ... 📌 Data synchronization jobs, migrating from one system to another, or creating analytics warehouses from SaaS application data.

📋 PROMPT

💡 EXAMPLE OUTPUT
Webhook Trigger (raw body mode) → Function node (HMAC verification: `crypto.createHmac('sha256', webhookSecret).update(rawBody).digest('hex') === extractedSignature`) → IF (valid signature) → Redis GET (check event ID) → IF (not processed) → Redis SET (event ID, processed=true, EX 86400) → Respond to Webhook (200 OK) → Execute Workflow trigger (async processing) → Switch (by event.type) → payment: update DB + send receipt → subscription_deleted: cancel access + send email. ```javascript

for (let attempt = 0; attempt <= maxRetries; attempt++) {
try {
const response = await fetch(url, options);

// Success
if (response.ok) return response.json();

// Don't retry client errors
if (response.status >= 400 && response.status < 500) {
if (response.status === 429) {
const retryAfter = response.headers.get('Retry-After') || 60;
await sleep(retryAfter * 1000);
continue; // Retry rate limits
}
throw new Error(`Client error: ${response.status}`);
}

// Retry server errors
if (response.status >= 500) {
if (attempt < maxRetries) {
const delay = Math.min(1000 × (2 ** attempt), 30000); // Max 30s
await sleep(delay);
continue;
}
}

throw new Error(`API error: ${response.status}`);

} catch (err) {
if (attempt === maxRetries) throw err;
await sleep(1000 × (2 ** attempt));
}
}
}

- Wait 60 seconds → try one request (half-open)
- If succeeds → close circuit (back to normal)
- If fails → stay open for another 60 seconds
